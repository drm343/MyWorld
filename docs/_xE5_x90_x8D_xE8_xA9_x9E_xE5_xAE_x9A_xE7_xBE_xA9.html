名詞定義

結構

文件中凡是指稱到結構的地方，一律指該結構本身。

註
程式中儘量不要直接用結構，而是使用結構的指標，只在生命周期極短可以使用 auto 的狀況下使用結構本身。

物件

文件中凡是指稱到物件的地方，一律指該結構的指標，注意這點跟 C 標準中所指的物件並不相同。

Access

這個名詞從 Ada 中取過來的，類似 C 的 Pointer，但擁有更多特性，凡是看到 type 後面加上 _Access 的 一律等同於前面所提物件在程式上的定義，這是為了避免使用太多 * 而採用的名稱慣例。

文件中單純只寫 Access 的情況則可以看成物件在記憶體中的實際位置。

註
要注意這是 C 不是 Ada，因此 Access 並沒有真的像 Ada 一樣有更多特性可以使用。

角色

若沒有特別註解，則文件中的「角色」皆代表 Character 物件。

角色狀態

若沒有特別註解，則文件中的「角色狀態」皆代表 Status 物件。

角色建立工廠

Character_Factory，生產角色卡來建立角色用的工具，將記憶體管理相關的工作放到這裡。

種族資料

基本結構等同於角色，可以看成角色原型，所有的角色都會根據種族來建立出新角色。

註
目前只有玩家沒有種族資料，因為還沒建立玩家使用的種族設定跟程式碼。

實體化

指從複製種族資料來產生新角色開始，到使用者完成後續處理設定的過程。

Game_Status

暫時的名稱，用來保存種族資料跟已經實體化的角色資料，未來會加上樓層相關資料。

圖形池

若沒有特別註解，則文件中的「圖形池」皆代表 Style_Pool 的物件。

地圖

若沒有特別註解，則文件中的「地圖」皆代表 Map_Type 物件，根據情況，未來可能跟樓層合併。

警告
關於地圖的結構，後續可能會修改。

Message_Box

用來顯示訊息的地方，預設使用 SDL 實作，原本在 Ada 中是打算 Generics 的方式來讓實作 跟定義分開，並且通用化，目前用 C 先不要考慮通用的問題。
